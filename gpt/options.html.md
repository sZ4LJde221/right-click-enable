# `options/options.html` の解説

役割

- ユーザーが常時有効化したいドメインの一覧を閲覧・操作するための UI を提供する静的 HTML。
- `options.js` を通じて `chrome.storage.sync` の `enabledDomains` を表示・編集する。

構造の要点

- スタイルは埋め込みでシンプルに整えられている（読みやすさ重視）。  
- `#domainList` にドメイン一覧を描画する仕組み。  
- `#status` は操作の成功/失敗をユーザーにフィードバックするための要素。

設計意図

- 直感的なトグルと削除ボタンでドメインを管理できる。  
- サーバー通信や複雑なクライアントサイドロジックはなく、ストレージを直接扱うシンプルな構成。

アクセシビリティ・注意点

- 大量のドメインが入ると一覧の扱いが問題になる（ページングや検索が将来的に必要）。  
- `options.js` 側で十分なエラーハンドリングを行う必要がある（`chrome.runtime.lastError` のチェックなど）。

要約

- `options` はユーザーがドメインを管理するための UI で、`enabledDomains` を操作することに特化している。
